"Snap"is a full-stack JavaScript development platform. It exposes two synchronized environments — a visual builder and a traditional code editor — where every change in one is instantly reflected in the other. The two are not separate tools. They are two interfaces over the same underlying codebase. 

 

The Visual Builder (webflow , figma) 

The visual builder is split into two tabs: Frontend and Backend. 

The Frontend tab is a drag-and-drop page builder in the style of Webflow. Components are placed, configured, and arranged visually. Layout, styling, and structure are handled through direct manipulation — no context switching into code required. The output is real HTML, CSS, and JavaScript. 

The Backend tab is a node-based canvas. Logic, API routes, database operations, and data flows are represented as connected nodes. Each node maps to real code — a function, a route handler, a query, a middleware. The canvas is not a diagram tool. It is a development environment. What you build there is what runs. 

Frontend-to-Backend Connection 

Frontend components and backend ( icepanel - visual reference and c4 models, c4 layering nodes setup) nodes connect through a shared tagging and binding system. A frontend element can be tagged and bound to a backend node — wiring a button to an API call, a data field to a query result, a form to a handler. The connection is explicit, inspectable, and reflected in code. This replaces the implicit mental mapping developers normally maintain between UI and server logic. 

The Code Editor 

The traditional code editor sits alongside the visual builder. It shows the actual JavaScript — frontend components, backend routes, utility functions, everything. Edits made in the visual layer appear here immediately. Edits made here reflect back in the visual layer immediately. Neither environment is the source of truth — they are synchronized representations of the same thing.
